# Ремонтная мастерская

## Введение

Современные ремонтные предприятия сталкиваются с необходимостью обработки значительных объёмов информации о клиентах, заказах, сотрудниках и оказываемых услугах. Внедрение специализированного программного обеспечения на базе реляционных баз данных позволяет структурировать эти сведения, ускорить документооборот и снизить риск ошибок, связанных с ручным ведением учёта.

Целью данной работы является разработка приложения «Ремонтная мастерская», обеспечивающего автоматизацию основных бизнес-процессов: приём и учёт заказов, ведение справочников услуг и исполнителей, регистрацию клиентов.

## 1 Проектирование базы данных

### 1.1 Выполнить проектирование БД

Проектирование базы данных выполнялось на основе технического задания. Для формализации структуры данных использовался аппарат ER-диаграмм.

В ходе анализа были выделены следующие ключевые сущности:
- Учётные записи пользователей (users)
- Клиенты мастерской (customers)
- Перечень услуг (work_types)
- Штатные сотрудники (performers)
- Журнал заказов (repair_orders)

## 3 Разработка интерфейса базы данных

### 3.1 Создание форм для ввода и редактирования данных

Для обеспечения комфортной работы пользователей с инфор��ационной системой спроектированы и реализованы следующие графические интерфейсы:

- Экранная форма «Вход в систему» — для идентификации пользователей
- Экранная форма «Регистрация» — для создания учётной записи клиента
- Экранная форма «Рабочее окно» — основной интерфейс с закладками
- Экранная форма «Работа с заказом» — для заполнения информации о заказе
- Экранная форма «Управление услугами» — для ведения справочника
- Экранная форма «Управление мастерами» — для ведения списка сотрудников
- Экранная форма «Управление клиентами» — для ведения базы заказчиков

### 3.2 Работа с данными: навигация, поиск, редактирование, фильтрация

Соединение с сервером PostgreSQL выполняется с использованием библиотеки Qt SQL.

### 3.3 Запросы

```sql
SELECT ro.id, ro.order_number, c.full_name AS client, 
       wt.name AS service, p.full_name AS master,
       ro.order_date, ro.deadline_date, ro.status, 
       ro.hours_spent, ro.total_cost, ro.description
FROM repair_orders ro
JOIN customers c ON ro.customer_id = c.id
JOIN work_types wt ON ro.work_type_id = wt.id
JOIN performers p ON ro.performer_id = p.id
ORDER BY ro.id;
```
